## §2.1 二層構造（実装設計）

### §2.1.1 [W11] の元の定義（再掲）

[W11] §2.1 では、方向制約波（$\kappa = 2$）の状態を以下のように定義する：

$$(\mathbf{k},\; \Delta\mathbf{k}) \in \mathbb{Z}^4 \times \mathbb{Z}^4$$

ここで：

- $\mathbf{k}$：**静的ラベル**（波の種類、[W8] の表 A に対応）
- $\Delta\mathbf{k}$：**動的状態**（エネルギー・運動量に対応）

この記法は接束 $\mathbb{Z}^4 \times \mathbb{Z}^4$ 上の点として状態を表現する。

### §2.1.2 D1 v2.0 データモデルへの写像

[D1] v2.0 §5.1 で定義された Particle 構造：

```
Particle = (
    σ   : SignVector,    -- 6 軸の振動周波数 [L⁻¹] / [L] / [1]
    V   : OneDWave[κ],   -- 各非零位置型軸の現在位相状態
    δV  : OneDWave[κ],   -- 1 ステップでの位相変位
    δt  : DeltaT,         -- 処理ステップ周期 [L]
    δδt : DeltaT,         -- 処理周期の変化（加速度効果）[L]
)
```

において、[W11] の二層構造 $(\mathbf{k}, \Delta\mathbf{k})$ は次の対応関係を持つ：

| [W11] の表記                   | D1 v2.0 における対応                             | 型                           | 備考                       |
| ------------------------------ | ------------------------------------------------ | ---------------------------- | -------------------------- |
| $\mathbf{k}$（静的ラベル）     | `σ : SignVector` の位置型 4 軸 (x, y, z, t) 成分 | `SpatialFreq × TemporalFreq` | 各軸の振動周波数 [L⁻¹]     |
| $\Delta\mathbf{k}$（動的状態） | `δV : OneDWave[κ]` の各要素の `Δθ₀` 成分         | `DeltaPhase[κ]`              | 1 ステップでの位相変位 [1] |
| （[W11] では暗黙）             | `V : OneDWave[κ]` の各要素の `θ₀` 成分           | `CyclePhase[κ]`              | 現在の位置位相 [1]         |
| （[W11] では暗黙）             | `δt : DeltaT`                                    | `L256`                       | 処理ステップ周期 [L]       |
| （[W11] では暗黙）             | `δδt : DeltaT`                                   | `L256`                       | 処理周期変化 [L]           |

すなわち：

> **$\mathbf{k}$ は σ.k_xyzt（4 つの振動周波数）に、$\Delta\mathbf{k}$ は δV[j].Δθ₀（4 つの位相変位）に対応する。**

[W11] が暗黙にしていた「位置位相 V」「処理ステップ δt」「加速度 δδt」が D1 v2.0 では明示的フィールドとして含まれる。

### §2.1.3 方向制約波（κ = 2）の具体的データ構造

[W11] における方向制約波（$\kappa = 2$、フェルミオン型）の状態を D1 v2.0 で書き下す：

```
Particle_κ2 = (
    σ : SignVector(
        k_x  : SpatialFreq,    -- 非ゼロ要素1（例：第1世代では k_x ≠ 0）
        k_y  : SpatialFreq,    -- ゼロ
        k_z  : SpatialFreq,    -- ゼロ
        k_t  : TemporalFreq,   -- 非ゼロ要素2（時間軸成分）
        R    : CurvatureR,     -- 0（フェルミオンは R = 0）
        Q    : ColorQ,         -- 種別（レプトン/クォーク）
    ),
    V  : OneDWave[2] = [
        wave_x : OneDWave(θ₀_x, A_x, ℓ_x),    -- x 軸方向の現在位相状態
        wave_t : OneDWave(θ₀_t, A_t, ℓ_t),    -- t 軸方向の現在位相状態
    ],
    δV : OneDWave[2] = [
        δwave_x : OneDWave(Δθ₀_x, ΔA_x, Δℓ_x),  -- x 軸の 1 ステップ位相変位
        δwave_t : OneDWave(Δθ₀_t, ΔA_t, Δℓ_t),  -- t 軸の 1 ステップ位相変位
    ],
    δt  : DeltaT,
    δδt : DeltaT,
)
```

[W11] の $\mathbf{k}$ は σ の非ゼロ位置型 4 軸成分（この例では $(k_x, 0, 0, k_t)$）として、$\Delta\mathbf{k}$ は δV の各要素の `Δθ₀` 成分として（$(\Delta θ_{0,x}, \Delta θ_{0,t})$）として表現される。

### §2.1.4 エネルギー・運動量との対応

[W11] では $\Delta\mathbf{k}$ を「動的状態（エネルギー・運動量に対応）」と記述しているが、**具体的にどのような演算でエネルギー・運動量相当の量を導出するか**は [W11] では明示されていない。本節でこれを明示する。

#### §2.1.4.1 D1 v2.0 における V と δV の役割

D1 v2.0 §5.2 の位相発展則：

$$V[j].\theta_0 \mathrel{+}= \delta V[j].\Delta\theta_0$$

において、$\delta V[j].\Delta\theta_0$ は **1 処理ステップで進む位相** である。

D1 v2.0 §6.1 の自由粒子の場合：

$$\delta V[j].\Delta\theta_0 = \sigma.k_{a(j)} \cdot \delta t$$

つまり：

$$\delta V[j].\Delta\theta_0 = (\text{周波数}) \times (\text{処理周期}) = \text{無次元位相} \quad [L^{-1}] \cdot [L] = [1]$$

#### §2.1.4.2 運動量に対応する量

標準物理では運動量 $p = \hbar k$（波数 × ℏ）。フレームワーク（c=1, G=1, ℏ なし）では波数 σ.k_x がそのまま運動量次元の量となる。

D1 v2.0 で運動量相当の量を取り出すには：

$$p_x^{\text{相当}} := \sigma.k_x \in \text{SpatialFreq} \quad [L^{-1}] = [M^{-1}]\text{（c=1 で）}$$

ただし、これは粒子の**静的な周波数値**であり、自由運動中は変化しない。**運動量変化**（散乱・力の作用）は σ.k 自体が変化することで表現される（相互作用イベントによる σ 更新）。

具体的演算として：

$$\Delta p_x^{\text{相当}} := \sigma'.k_x - \sigma.k_x \quad \text{（相互作用前後の周波数差）}$$

#### §2.1.4.3 エネルギーに対応する量

標準物理ではエネルギー $E = \hbar \omega$（角振動数 × ℏ）。フレームワークでは時間軸の周波数 σ.k_t がエネルギー次元相当：

$$E^{\text{相当}} := \sigma.k_t \in \text{TemporalFreq} \quad [L^{-1}] = [M^{-1}]\text{（c=1 で）}$$

ここで重要な点：**エネルギーと運動量は σ の異なる軸成分として同じ次元 $[L^{-1}]$ で表現される**。これは特殊相対論の 4 元運動量 $(E/c, p_x, p_y, p_z)$ と同型構造である。

質量殻条件は D1 v2.0 §3.4 に従い：

$$\sigma.k_t^2 = \sigma.k_x^2 + \sigma.k_y^2 + \sigma.k_z^2 + m^2$$

として整数値間の関係で記述される。

#### §2.1.4.4 では δV はエネルギー・運動量か？

[W11] が「$\Delta\mathbf{k}$ ＝ 動的状態（エネルギー・運動量に対応）」と書いているのは、**正確ではない**。D1 v2.0 で精密に定義し直すと：

| [W11] の主張                                         | D1 v2.0 での精密化                                                       |
| ---------------------------------------------------- | ------------------------------------------------------------------------ |
| $\mathbf{k}$ ＝ 静的ラベル                           | σ.k_xyzt の値そのもの                                                    |
| $\Delta\mathbf{k}$ ＝ 動的状態（エネルギー・運動量） | **δV[j].Δθ₀（位相変位）** ではなく、**σ.k 自体**がエネルギー・運動量次元 |

正しくは：

> **エネルギー・運動量に対応する量は σ.k_t（エネルギー）と σ.k_xyz（運動量）であり、δV[j].Δθ₀ はこれらに処理周期 δt を掛けた「1 ステップで進む位相」である。**

つまり：

$$\delta V[j].\Delta\theta_0 = \sigma.k_{a(j)} \cdot \delta t = (\text{エネルギー or 運動量}) \times (\text{時間相当})$$

これは標準物理での **作用量** $S = E \cdot t$（または $p \cdot x$）と同型構造である。位相が累積した値 $V[j].\theta_0$ は、量子力学における作用 $S/\hbar$（位相因子 $e^{iS/\hbar}$）に対応する。

### §2.1.5 まとめ：[W11] §2.1 の D1 v2.0 仕様

[W11] §2.1 の二層構造を D1 v2.0 で完全に書き直すと：

**状態の表現**：

$$\text{Particle} = (\sigma, V, \delta V, \delta t, \delta\delta t)$$

ここで：

- $\sigma$（SignVector）：6 軸の振動周波数 ＝ [W11] の $\mathbf{k}$ を 6 軸に拡張したもの。エネルギー・運動量・スケール量を全て含む
- $V$（OneDWave[κ]）：各非零位置型軸の現在位相 ＝ [W11] では暗黙だった、波の現在状態
- $\delta V$（OneDWave[κ]）：1 ステップでの位相変位 ＝ σ.k と δt の積、[W11] の $\Delta\mathbf{k}$ と関連するが厳密には別物
- $\delta t$（DeltaT）：処理ステップ周期 ＝ 1 ステップの「時間相当量」
- $\delta\delta t$（DeltaT）：処理周期の変化 ＝ 加速度効果

**エネルギー・運動量の取り出し**：

$$E^{\text{相当}} = \sigma.k_t, \quad p_i^{\text{相当}} = \sigma.k_i \quad (i \in \{x, y, z\})$$

**位相の累積（作用相当）**：

$$V[j].\theta_0 \mathrel{+}= \sigma.k_{a(j)} \cdot \delta t \quad (\text{各ステップ})$$

これにより、[W11] が「動的状態」と曖昧に表現していた量が、D1 v2.0 のフィールドに**厳密に対応付けられた**。

## §2.2 基本演算 `step(p)`：1 粒子の時間発展

### §2.2.0 引数の基本ルール（5 原則）

本論文のすべての処理関数（本節の `step(p)` および §3 以降の相互作用関数）は、以下の 5 つの基本ルールに従う。これらは**処理関数の正当性を保証する制約**であり、満たされない関数は仕様として認められない。

**ルール 1（null 透過性）**: INPUT が `null` の場合、OUTPUT も `null` を返す。

```
function f(p : Particle | null) → Particle | null:
    if p == null:
        return null
    ...
```

これは粒子の消滅・不在を表現する基本機構である。例えば吸収過程で「ゲージボソンが消滅する」が、消滅した粒子に対して関数を呼んでも `null` が返るため、後続処理が破綻しない。

**ルール 2（in-place 更新による保存量維持）**: INPUT をそのまま更新して OUTPUT として返す。新しい Particle を生成・破棄しない。これにより保存量は構造的に維持される（何も増減しないから自明）。

**ルール 3（型整合性）**: INPUT の型が要件を満たさない場合は、コンパイル時または実行時に型エラーで弾く。silent failure は許容しない。

**ルール 4（INPUT/OUTPUT 入れ替え可能性）**: INPUT と OUTPUT を入れ替えても、関数の構造的整合性が保たれる。これは処理が「片方向の破壊的変換」ではなく「対称な状態変換」であることを意味する。

**ルール 5（時間反転による可逆性）**: `δt` の符号を反転して同じ関数を再適用すると、処理が完全に巻き戻る。すなわち：

```
step(step(p, δt = +δt₀), δt = -δt₀) == p   （元の状態に戻る）
```

これは情報の非可逆的損失が起きないことを保証する（[W3] の情報保存要件）。

### §2.2.1 基本演算の定義

すべての粒子の時間発展は次の単一関数で記述される：

```
function step(p : Particle | null) → Particle | null:
    -- ルール 1: null 透過
    if p == null:
        return null
    
    -- 各非ゼロ位置型軸 j について位相を進める
    for j in range(p.κ):
        p.V[j].θ₀ += p.σ.k_{a(j)} · p.δt
    
    return p
```

**INPUT**: `Particle | null` 1 つ  
**処理**: `p` が null なら null 返却、そうでなければ V[j].θ₀ を更新（整数加算と乗算のみ）  
**OUTPUT**: 同じ `p`（更新済み）または `null`

`σ`, `δV`, `δt` は `step(p)` では一切更新されない。これらは粒子の状態として保持されるだけで、本関数の処理対象外。

### §2.2.2 5 原則の検証

**ルール 1（null 透過）**: 関数の最初で明示的に処理 ✓

**ルール 2（in-place 更新）**: V[j].θ₀ を直接更新、新しい Particle 非生成 ✓

**ルール 3（型整合）**: σ.k_{a(j)} は `Linv256`、δt は `L256`、積は `Scalar256 [1]`、これに `CyclePhase` を加算（mod P/2）— D1 v2.0 §2.3 の規則に準拠 ✓

**ルール 4（INPUT/OUTPUT 入れ替え可能性）**: 関数は `p` を読み書きする対称構造 ✓

**ルール 5（時間反転による可逆性）**:

```
step 1 回目（δt = +δt₀）:
    V[j].θ₀ → V[j].θ₀ + σ.k · δt₀

δt の符号反転（δt → -δt₀）

step 2 回目（δt = -δt₀）:
    V[j].θ₀ → V[j].θ₀ + σ.k · δt₀ + σ.k · (-δt₀) = V[j].θ₀
```

完全に元に戻る ✓

**5 原則すべてを満たす**ことが厳密に確認できる。

### §2.2.3 この関数だけで全てが処理できる

**`step(p)` の処理内容は一切変わらない**。粒子の挙動の違い（静止・等速直線運動）は、すべて **INPUT の Particle に与えられている `σ` と `δt` の値の違い**として表現される。

```
複数粒子の時間発展:

function step_all(particles : List<Particle | null>):
    for each p in particles:
        step(p)
```

各粒子は独立に `step(p)` を実行する。粒子間に相互作用がない場合、これで全ての自由運動が記述される。

以下、具体的な引数例を示す。

### §2.2.4 例：等速測地線運動

x 方向の測地線に沿って一定速度で移動する電子型粒子（$\kappa = 2$）の引数表現：

```
p = Particle(
    σ = SignVector(
        k_x = +k₀,        -- x 軸の振動周波数（運動量相当、一定）
        k_y = 0,
        k_z = 0,
        k_t = +k_t₀,      -- t 軸の振動周波数（エネルギー相当、一定）
        R   = 0,
        Q   = 0,
    ),
    V = [
        wave_x = OneDWave(θ₀=θ₀_x_init, A=A_x, ℓ=ℓ_x),
        wave_t = OneDWave(θ₀=θ₀_t_init, A=A_t, ℓ=ℓ_t),
    ],
    δV = [
        δwave_x = OneDWave(Δθ₀ = k₀ · δt₀, ΔA=0, Δℓ=0),
        δwave_t = OneDWave(Δθ₀ = k_t₀ · δt₀, ΔA=0, Δℓ=0),
    ],
    δt  = δt₀,           -- 処理周期（一定）
)
```

**重要なポイント**：

- `σ.k_x ≠ 0`、`σ.k_y = σ.k_z = 0` → x 方向のみの移動
- `δt = δt₀`（一定） → 等速運動
- `δV[*].ΔA = 0`, `δV[*].Δℓ = 0` → 振幅・広がりも変化しない → 形状不変

これを `step(p)` に渡すたびに：

```
V[x].θ₀ += k₀ · δt₀     -- x 位相が一定量進む
V[t].θ₀ += k_t₀ · δt₀   -- t 位相が一定量進む
他の V は変化しない（σ.k = 0 なので）
```

x 方向に等速直線運動する粒子が、整数演算だけで完全に表現される。

### §2.2.5 `step(p)` で表現できる運動の範囲（厳密）

**`step(p)` は測地線方向の等速運動のみを表現する**。これは GR の自由粒子原理（外力のない粒子は測地線に従う）に厳密に整合する設計である。

| 運動の種類     | `σ` の条件               | `δt` の条件 | 物理的意味               |
| -------------- | ------------------------ | ----------- | ------------------------ |
| 静止           | 全ゼロ                   | 任意        | 静止粒子                 |
| 等速測地線運動 | 特定軸が非ゼロ・他はゼロ | 一定        | 自由粒子の等速直線運動   |

**処理関数 `step(p)` は一切変化しない**。2 種類すべて、INPUT の Particle 状態の違い（σ と δt の値）だけで表現される。

#### `step(p)` で表現できないもの

以下は `step(p)` 単独では原理的に実装不可能：

- **加速度運動**：δt が変化する必要があるが、これはルール 5（可逆性）と保存則を破る
- **振動**（調和振動子など）：復元力が必要
- **円運動・曲線運動**：向心力・横向きの力が必要
- **任意の曲がった軌道**：測地線から外れるには外力が必要

これらは GR の原理上、**外力（相互作用）を必要とする運動**である。フレームワークでは：

- 粒子間相互作用（メッシュ点共有時の σ・δV の変化、§3 以降）
- 背景場との非自明な結合（Mesh の縦波モードによる σ.k 実効値の変調）

として §3 以降で扱う。

#### なぜ加速度運動を `step(p)` で扱えないか（厳密検証）

仮に `step(p)` 内で δt を更新する仕様（`δt += δδt`）を考える：

```
step 1 回目（δδt = +a₀）:
    δt → δt₀ + a₀
    V[j].θ₀ → V[j].θ₀ + σ.k · (δt₀ + a₀)

δt の符号反転（δt → -(δt₀ + a₀)）

step 2 回目（δδt = +a₀）:
    δt → -(δt₀ + a₀) + a₀ = -δt₀
    V[j].θ₀ → V[j].θ₀ + σ.k·(δt₀ + a₀) + σ.k·(-δt₀) = V[j].θ₀ + σ.k·a₀
```

**ルール 5（可逆性）が破れる**：V も δt も元の値に戻らない。

さらに、単一粒子で δt が変化することは、**何の相手とも交換せずに固有時の進み方が変わる**ことを意味する：

- σ.k（運動量・エネルギー）は変わらない
- しかし固有時の進み方が外因なく変わる
- これは**エネルギー・運動量保存則の暗黙の破れ**を意味する（保存則の整合性が失われる）

したがって、加速度効果を表現する変数（δδt 等）は **基本演算 `step(p)` には含めない**。これは設計上の必然である。

#### Particle 構造から δδt は削除される

D1 v2.0 §5.1 では `Particle` に `δδt : DeltaT` フィールドを含めていたが、本論文の解析により：

- `δδt` を `step(p)` で処理すると 5 原則（特にルール 5）を破る
- 加速度効果は単一粒子では表現不可能（保存則と整合しない）

したがって **δδt は Particle 構造から削除すべき**。修正版：

```
Particle = (
    σ   : SignVector,
    V   : OneDWave[κ],
    δV  : OneDWave[κ],
    δt  : DeltaT,
)
```

加速度効果は §3 以降の相互作用処理（粒子間または粒子-Mesh 間）の中で、保存則を満たす形で **σ や δt が変化** することで表現される。

D1 v2.1 として正式に修正版を公開予定。

#### 「曲げられない」「加速できない」のは設計として正しい

`step(p)` で軌道が曲がったり加速したりしたら、**外力なしで運動量・エネルギーが変化**することになり、保存則と可逆性が破れる。GR の自由粒子原理：

> **自由粒子は測地線に従う。測地線から外れるには「力」（相互作用）が必要。**

`step(p)` はこの原理を厳密に実装している。これは欠陥ではなく、**物理原理に従う正しい設計**。

#### 中心投影フレームワークでは「測地線」は格子に内在

中心投影された主観空間 $S^4(R_0)$ は曲がった空間。その上の整数格子（メッシュ）も曲がっている。

> **格子上を「直線的に進む」こと自体が、実空間（連続極限）では測地線（曲線）になる**

例：球面上を「真っ直ぐ歩く」と大円（測地線）を描くのと同じ。

つまり `step(p)` で σ.k と δt の積として位相が進む処理は、**自動的に測地線運動を実現している**。粒子は格子上を「真っ直ぐ」進むが、それが曲がった背景空間では測地線として現れる。**「曲げる」「加速する」ための追加機構は不要**——背景の曲率は格子構造に内蔵済み。

### §2.2.6 §2.2 まとめ

- **基本演算は `step(p)` 1 つ**：V[j].θ₀ のみ更新
- **5 原則すべてを満たす**：null 透過・in-place 更新・型整合・対称性・可逆性
- **保存量は構造的に維持される**：何も生成・消滅しない、σ と δt は変わらない
- **`step(p)` は等速測地線運動のみ表現する**：静止または等速直線運動
- **加速度・曲線運動・振動は外力が必要 → §3 以降の相互作用機構で扱う**
- **D1 v2.0 の `δδt` フィールドは削除する**（D1 v2.1 で修正）

[W11] §2.2 の定義 2.1 の「→」のうち、相互作用 **しない** 部分（自由運動）はこれで完全に展開された。相互作用する部分（軌道を曲げる・σ や δt を変化させる）は §3 以降で扱う。

## 各作業で困難だった点

### 困難1：69ページのJHEP論文（2302.11695）の翻訳
- 全文翻訳は時間がかかりすぎるため、要点抜粋訳に切り替え
- Abstract、Introduction、Section 2.1（定義）、Section 6（結論）を中心に翻訳

- これで論文の中心メッセージは把握できているが、詳細な計算過程は追えていない

### 困難2：Pseudo Entropy の複素数値の物理的意味
- 数学的な定義は明確（遷移行列の von Neumann エントロピー類似量）
- しかし「複素数値が何を測っているか」は私には分からなかった
- ただしこれは**講演者本人も「未解決問題」と明示的に挙げている**ため、私の理解不足が個人的なものではないと判断

### 困難3：弱値（weak value）と Pseudo Entropy の接続
- スライド #20 で重要な接続として提示されている
- Aharonov-Albert-Vaidman 1988 という古典的な参考文献
- 量子論の特殊な道具で、私の知識では完全には追えなかった
- 「PE は Modular Operator の弱値として書ける」という主張までは理解できたが、その量子情報的含意は把握できていない

### 困難4：「時間の創発」という概念の物理的意味
- 「空間が量子もつれから創発する」は Van Raamsdonk 2010, Swingle 2012 以来確立されつつあるが、「時間の創発」は遥かに難しい
- 私の理解では、これは「量子論で時間を演算子ではなく外部パラメータとして扱う」伝統的な問題と接続している
- 比喩を書こうとしたが、「動画と静止画の違い」程度しか書けず、本質的な比喩には届かなかった

### 困難5：dS/CFT の双対 CFT が「虚数中心電荷」を持つこと
- ユニタリーCFT に慣れた立場からは病的に見える
- 「Liouville CFT で記述できる」という主張があるが、私には数学的に追えなかった
- 業界自身も「非ユニタリーCFT の数学的整理」が遅れていることを認めている

## CG実装者の視点で気づいた構造的特徴

**注意**：以下は「実装者として読んでこう感じた」というレベルの感想であり、業界批判ではありません。私の理解には誤解の可能性があります。

### 気づき1：複素数値は CG 屋にはむしろ自然
- 物理学者は「複素数値の物理量」に対して若干の違和感を持つようだが、CG 屋には「フィルタのゲイン」「フーリエ変換」「振幅と位相の同時表現」など馴染みがある
- 「複素数 = 病的」ではなく「複素数 = 振幅と位相を1つの数で書く便利な記法」と思える
- 物理側で複素数値が出ること自体は、CG 屋的視点からは**取り扱いやすそう**に見える

### 気づき2：Wick 回転は「座標変換の一種」
- 物理学者は Wick 回転（実時間 ↔ 虚時間）を特殊な操作と捉えるが、CG/シミュレーション業界では「**時間軸を別のパラメータに置き換えて問題を解く**」のは普通のテクニック
- 例：レンダリング方程式を「直接解く」ではなく「散乱イテレーションで解く」、流体シミュレーションで「圧力 Poisson 方程式」を別の枠組みで解く、など
- 抵抗感は少ない

### 気づき3：テンソルネットワークは CG のシーングラフに似ている
- MERA、HaPPY model のような Tensor Network は、CG のシーングラフ・ノードベース構造と類似
- 「**ローカル演算とつなぎ合わせで全体を記述**」という発想は CG では当たり前
- ただし量子的確率振幅の足し上げは CG にはない概念

### 気づき4：「観測者を含む系」の組み込みは CG 屋には未解決問題ではない
- CG では「カメラ（観測者）の位置」を**最初から座標系に組み込む**のが当然
- 物理学では「観測者をどう量子論に組み込むか」が長年の問題
- 量子論のこの困難は CG 屋的視点からは**馴染みのない種類の問題**

### 気づき5：「経路積分最適化が重力を生む」というのは CG 的に共感しやすい
- 経路積分最適化（path-integral optimization）= **計算コスト最小化**
- これが重力を生む、というのは CG 屋的に「**最小作用原理**」「**最短経路探索**」「**Voronoi図のような最適化幾何**」を連想する
- Penrose のスピンネットワークなど、過去の量子重力アプローチと通じる

### 気づき6：論文・スライドの「強調の使い方」が CG プレゼンと共通
- 高柳先生のスライドは「赤字 + 太字 + 矢印 + 図」の組み合わせが洗練されている
- CG 業界のテックトーク（GDC, SIGGRAPH のような場）でのプレゼン作法と類似
- 「Question / The answer is...」という問答形式、「Ver.1 → Ver.2 → Ver.3」というバージョン進化のメタファー、最後を**たった2つの未解決問題**に絞る構成――これらは聴衆を引き込む技巧として、業界を超えて共通

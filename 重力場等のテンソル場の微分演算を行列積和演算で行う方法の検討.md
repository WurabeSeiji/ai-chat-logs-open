# 研究レポート（続編）
## 重力場等のテンソル場の微分演算を行列積和演算（FMA）で行う手法の検討
---

## 1. はじめに

前報『閉じた正曲率時空におけるテスト粒子運動のGPU効率的シミュレーション技法（テンソル演算版）』[1] では、曲率を持つ時空における測地線運動を、事前計算された4次元テンソルのルックアップと線形補間（Lerp）によって超高速に近似計算する手法を提示した。

本稿では、その背後にある物理的基盤を確立するため、一般相対性理論における微分幾何学の諸演算が、GPU（Graphics Processing Unit）アーキテクチャ上の**行列積和演算（FMA: Fused Multiply-Add）パイプライン**にいかに還元可能かを構造的に整理する。

本研究の主な貢献は以下の3点である。
1. 一般相対論のテンソル演算の、GPU最適化可能なFMAパイプラインへの完全分解
2. 実装における階層的な計算グラフ（データ依存性）と計算量の明示
3. 前報の「平坦空間近似（速度優先）」モデルに対する、「曲率演算モデル（精度優先）」の理論的提供と、両者のトレードオフ評価基盤の構築

## 2. 演算概念のマッピングと誤差オーダー

理論物理学における連続テンソル演算を、計算機上の離散アーキテクチャ（Regge (1961) [3] などの離散幾何アプローチ）に対応させるため、次のような概念と精度の変換を行う。

*   **場（Field）** → **配列・テクスチャ（N-Dimensional Array / Texture）**
*   **微分（Derivative）** → **差分ステンシル（例: 2次精度中央差分。空間量子化誤差 $\mathcal{O}(\Delta x^2)$ を伴う）**
*   **非線形項（Non-linear term）** → **行列積（Matrix Multiplication）**
*   **縮約（Contraction）** → **反復加算・ドット積（Reduce / Dot Product）**

この変換により、幾何学的な微積分操作を「計算機上の算術演算とメモリアクセス」へと分解し、数値解析的な誤差評価が可能となる。

## 3. アインシュタイン方程式のFMA計算グラフ化

時空の曲率および重力場の方程式の計算プロセスを、実装依存のデータ依存性グラフとして分解する。各方程式は Misnerら (1973) [2] の定義に準拠し、離散格子上の $4 \times 4$ 行列テンソル $g_{\mu\nu}$ を起点とする。

計算グラフの全体構造は以下の依存性（データフロー）を持つ：
$[ g_{\mu\nu} ] \xrightarrow{\text{Stencil}} [ \partial g, g^{\mu\nu} ] \xrightarrow{\text{FMA}} [ \Gamma^\lambda_{\mu\nu} ] \xrightarrow{\text{Stencil} + \text{FMA}} [ R^\rho_{\sigma\mu\nu} ] \xrightarrow{\text{Reduce}} [ R_{\mu\nu}, R ]$

### 3.1. 計量テンソルの空間微分の差分化
偏微分 $\partial_\mu g_{\alpha\beta}$ は、中央差分を用いて離散化される。
$$ \frac{\partial g_{\alpha\beta}}{\partial x^\mu} \approx \frac{g(x + \Delta x^\mu)_{\alpha\beta} - g(x - \Delta x^\mu)_{\alpha\beta}}{2\Delta x^\mu} + \mathcal{O}(\Delta x^2) $$
隣接タクセル（Texel）の線形結合であり、メモリアクセス主体のステンシル演算となる。

### 3.2. 計量テンソルの逆行列算定
反変計量テンソル $g^{\mu\nu}$ は空間各点の $g_{\mu\nu}$ の $4 \times 4$ 逆行列計算である。

### 3.3. クリストッフェル記号の算出
$$ \Gamma^\lambda_{\mu\nu} = \frac{1}{2} g^{\lambda\sigma} \left( \partial_\mu g_{\nu\sigma} + \partial_\nu g_{\mu\sigma} - \partial_\sigma g_{\mu\nu} \right) $$
**計算量評価**: 空間の各点において、逆行列とのテンソル積（$\sum g^{\mu\nu} \cdot \partial g$）を評価するため、多重ループ内で約64回の最適化されたFMA命令を消費する密な演算ノードとなる。

### 3.4. 曲率テンソルの算出
$$ R^\rho_{\sigma\mu\nu} = \partial_\mu \Gamma^\rho_{\nu\sigma} - \partial_\nu \Gamma^\rho_{\mu\sigma} + \Gamma^\rho_{\mu\lambda}\Gamma^\lambda_{\nu\sigma} - \Gamma^\rho_{\nu\lambda}\Gamma^\lambda_{\mu\sigma} $$
第一項・第二項は $\Gamma$ に対する差分ステンシルである。第三項・第四項の $\Gamma \Gamma$ は非線形項であるが、計算機上は $4 \times 4$ 行列同士の積の組み合わせ（約256要素の算出）であり、極めてFMA密度の高いアレイ演算となる。

### 3.5. 縮約処理とアインシュタインテンソルの構築
リッチテンソル $R_{\mu\nu}$ およびスカラー曲率 $R$ への縮約はドット積（Reduce加算）であり、最終的な $G_{\mu\nu}$ の算出に至るまで完全に算術的なFMAによって処理される。

## 4. GPUアーキテクチャとの親和性とボトルネック分析

一般相対論の微分方程式は、本質的に「高いFMA密度を持つ並列計算グラフ」へと還元されるため、現代のGPUと極めてハードウェア親和性が高い。実装にあたっては以下の特性を考慮する。

1. **FMAスループット**: 曲率計算の非線形項（行列積）はALU負荷が高いため、GPUのTensor Coreや広範なベクタレジスタ構成に極めて適している。
2. **メモリアクセスとKernel Fusion**: 各ステップ（$g \to \Gamma \to R$）の中間テンソルをすべてVRAMに書き出すと、深刻なメモリ帯域幅律速（Memory Bound）に陥る。実運用では Shared Memory の活用やレジスタ内での全方程式計算（Kernel Fusion）が不可欠である。

## 5. 実証実験設計：平坦近似と曲率計算のトレードオフ（次報への展望）

前報 [1] で示した手法は、計算負荷の高い曲率計算を回避し、マッピング関数による事前計算と線形補間（Lerp）のみを行う**「平坦空間近似・超高速モデル（Mode A）」**である。
対して本稿のFMAパイプラインは、メッシュ上の曲率を直接計算する**「曲率計算・物理厳密モデル（Mode B）」**である。

今後の実装フェーズにおいて、以下の定量評価を実施する。
*   **パフォーマンス評価**: メッシュサイズ（$N^3$）に対するスループット（GFLOPSや実行時間）の計測。GPU実装の効果を実証する。
*   **精度と破綻の評価**: $\Delta x \to 0$ に対する誤差収束（$\mathcal{O}(\Delta x^2)$）の確認。また、強重力場においてMode A（近似）が物理的に破綻し、Mode B（本手法）が正しい曲率軌道を描く限界ラインをエラー関数 $||R_{num} - R_{exact}||$ として可視化する。

## 6. 今後の課題（数値解析上の要請）

実用的なシミュレータとして完成させるためには、以下の物理制約を導入する必要がある。
1. **CFL条件の適用**: 時間発展シミュレーションを行う場合、情報伝播速度に基づく制約 $\Delta t \le \Delta x / c$ を満たさなければならない。
2. **ゲージ条件と数値安定性**: 単純な中央差分の連続は高周波ノイズを増幅させる。長時間積分に際しては、BSSN定式化や調和ゲージ（Harmonic Gauge）といった数値相対論の標準スキームの導入が必要である。

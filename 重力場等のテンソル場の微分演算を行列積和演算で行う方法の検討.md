# 研究レポート（続編）
## 重力場等のテンソル場の微分演算を行列積和演算（FMA）で行う手法の検討
---

## 1. はじめに

前報『閉じた正曲率時空におけるテスト粒子運動のGPU効率的シミュレーション技法（テンソル演算版）』[1] では、曲率を持つ時空における測地線運動を、事前計算された4次元テンソルのルックアップと線形補間（Lerp）によって超高速に近似計算する手法を提示した。

本稿ではその背後にある物理的基盤を確立するため、一般相対性理論における微分幾何学の連続体方程式が、GPU（Graphics Processing Unit）上の**行列積和演算（FMA: Fused Multiply-Add）パイプライン**にいかに還元可能かを構造的に整理する。

本研究の新規性は、「テンソル演算がFMA演算に分解可能である」という数学的自明性の指摘に留まらない。本質的な貢献は、**一般相対論の非線形・高階テンソル計算を、GPUアーキテクチャ（メモリ帯域幅とFMA密度）に最適化された計算グラフとして再構築し、Arithmetic Intensity（演算強度）の観点から計算機物理シミュレーションを極限まで最適化可能であることを示した点**にある。
なお、本手法における曲率展開は一切の近似を含まず、一般相対論の標準的定義（Misnerら [2]）に完全準拠している。

## 2. 演算概念のマッピングと誤差オーダー

理論物理学における連続テンソル演算を、計算機上の離散アーキテクチャ（Regge (1961) [3] などの離散幾何アプローチに類比）に対応させるため、次のような概念と精度の変換を行う。

*   **場（Field）** → **配列・テクスチャ（N-Dimensional Array / Texture）**
*   **微分（Derivative）** → **差分ステンシル（例: 2次精度中央差分。空間量子化誤差 $\mathcal{O}(\Delta x^2)$ を伴う）**
*   **非線形項（Non-linear term）** → **行列積（Matrix Multiplication）**
*   **縮約（Contraction）** → **反復加算・ドット積（Reduce / Dot Product）**

## 3. アインシュタイン方程式の計算グラフ化とFLOP評価

時空の曲率方程式の計算プロセスを、実装依存のデータフロー（計算グラフ）として分解する。
全体の依存性は以下のように構成され、グリッド点数を $N^3$ としたときの全体計算量は $\mathcal{O}(N^3 \times 4^4)$ にスケーリングされる。

$$[ g_{\mu\nu} ] \xrightarrow{\text{Stencil}} [ \partial g, g^{\mu\nu} ] \xrightarrow{\text{FMA}} [ \Gamma^\lambda_{\mu\nu} ] \xrightarrow{\text{Stencil} + \text{FMA}} [ R^\rho_{\sigma\mu\nu} ] \xrightarrow{\text{Reduce}} [ R_{\mu\nu}, R ]$$

### 3.1. 計量テンソルの空間微分の差分化
偏微分 $\partial_\mu g_{\alpha\beta}$ は、中央差分を用いて離散化される。
$$ \frac{\partial g_{\alpha\beta}}{\partial x^\mu} \approx \frac{g(x + \Delta x^\mu)_{\alpha\beta} - g(x - \Delta x^\mu)_{\alpha\beta}}{2\Delta x^\mu} + \mathcal{O}(\Delta x^2) $$

### 3.2. 計量テンソルの逆行列算定
空間各点の計量 $g_{\mu\nu}$ の $4 \times 4$ 逆行列として反変計量テンソル $g^{\mu\nu}$ を算出する。
**FLOPカウント**: $4 \times 4$ 行列の逆行列計算は解析的に展開可能であり、余因子行列等を用いた単純なFMA演算として定数コスト（約100〜200 FLOPs程度）で評価可能である。

### 3.3. クリストッフェル記号の算出
$$ \Gamma^\lambda_{\mu\nu} = \frac{1}{2} g^{\lambda\sigma} \left( \partial_\mu g_{\nu\sigma} + \partial_\nu g_{\mu\sigma} - \partial_\sigma g_{\mu\nu} \right) $$
**FLOPカウント**: 空間各点において逆行列のテンソル積 $\sum g^{\mu\nu} \cdot \partial g$ を評価するため、3階の多重ループ内で約 **$4 \times 4 \times 4 = 64$ FMAs**（積和演算命令）を消費する。

### 3.4. 曲率テンソルの算出
$$ R^\rho_{\sigma\mu\nu} = \partial_\mu \Gamma^\rho_{\nu\sigma} - \partial_\nu \Gamma^\rho_{\mu\sigma} + \Gamma^\rho_{\mu\lambda}\Gamma^\lambda_{\nu\sigma} - \Gamma^\rho_{\nu\lambda}\Gamma^\lambda_{\mu\sigma} $$
第一項・第二項は $\Gamma$ に対する差分演算である。一方、第三項・第四項の $\Gamma \Gamma$ は非線形であるが、実装上は $4 \times 4$ 行列同士の積の反復結合である。
**FLOPカウント**: 空間各点で最大 **$4^5 = 1024$ FMAs** オーダーの演算を要する、極めてFMA密度の高いノードとなる。

### 3.5. 縮約処理とアインシュタインテンソルの構築
続くリッチテンソル $R_{\mu\nu}$ およびスカラー曲率 $R$ の算出に係る縮約処理はドット積（Reduce加算）であり、最終テンソル $G_{\mu\nu}$ に至るまで純粋な算術FMAパイプラインとして完結する。

## 4. GPUアーキテクチャへの最適化（演算強度とKernel Fusion）

従来の数値シミュレーション実装においては、微分計算やテンソル代数を個別に評価しVRAMへ逐次退避するため、非構造的なメモリアクセスが発生しメモリ帯域幅律速（Memory Bound）に陥りやすかった。
対して、本手法で提示したFMA計算グラフは、GPUアーキテクチャに対して以下の決定的な優位性を持つ。

1. **高い演算強度（Arithmetic Intensity）**:
   例えば入力データである計量テンソル $g_{\mu\nu}$ のメモリアクセスは空間1点あたり 16 Float（64 Bytes）に過ぎない。差分ステンシルにより近傍格子点へのメモリアクセスが必要となるが、GPUのShared Memory（共有メモリ）を用いたタイル化によって効率的にデータ再利用が可能である。
   その結果、読み込まれた少量のデータから $\Gamma$ 算出の64 FMAsやリーマン曲率算出の約1024 FMAs以上といった極めて濃密な演算が発生する。この高い Arithmetic Intensity を持つため、適切なメモリ最適化を施した場合には、並列計算ユニットの稼働率を最大化する「Compute Bound」な挙動を示すことが期待される。
2. **カーネルフュージョン（Kernel Fusion）の適用**:
   演算強度が高い計算グラフであるため、計量 $g_{\mu\nu}$ のグローバルメモリからの読み込みを一回行うだけで、中間変数（$\partial g, \Gamma, R$）をすべてレジスタやShared Memory（共有メモリ）内に留置したまま計算を進め、最終結果（$G_{\mu\nu}$ 等）のみをVRAMに書き出す最適化（Kernel Fusion）が完全に適用可能である。

## 5. 実証実験設計：平坦近似（Mode A）と厳密曲率計算（Mode B）のトレードオフ

前報 [1] は、演算負荷の高い曲率計算をスキップし、局所慣性近似と線形補間（Lerp）のみを用いる**「平坦空間近似・超高速モデル（Mode A）」**である。
対して本稿のFMAパイプラインは、メッシュ上の曲率（2階効果）を直接・厳密に計算する**「曲率計算・物理厳密モデル（Mode B）」**と定義できる。

両者のトレードオフは以下のように概念化（図解）できる。

```text
  [計算スループット] ↑
                   │    Mode A (Lerp)
    (高速・近似)    │    ※ 前報の手法。超高速だが強曲率領域では破綻
                   │
                   │                           Mode B (FMA Pipeline)
    (重負荷・正確)  │───────────────────────── ※ 本稿の手法。重力曲率を動的に厳密計算
                   └─────────────────────────────────────→ [物理的精度]
```

今後の実装フェーズにおいて、以下の定量指標（ベンチマーク）を測定し両者を比較評価する。
*   **計算速度のスケーリング**: メッシュサイズ（$N^3$）に対するスループット（GFLOPSやミリ秒）の比較。
*   **精度の検証（物理的破綻点の特定）**: 強重力場モデルにおいて、Mode A（近似）が一般相対論から外れて破綻する境界を、誤差ノルム $||R_{num} - R_{exact}||$ のヒートマップとして定量化し、Mode Bの優位性を実証する。

## 6. 今後の課題（数値安定性の要請）

本稿のFMAスキームを動的な時間積分（シミュレータ）へと発展させるためには、以下の物理的制約が必須となる。
1. **CFL条件**: 情報伝播速度に基づく制約 $\Delta t \le \Delta x / c$ を満たすステップ幅の管理。
2. **ゲージ条件設定**: 長時間積分に伴う高周波誤差の増幅を防ぐため、数値相対論の標準スキームであるBSSN形式や調和ゲージ（Harmonic Gauge）といった制約方程式の組み込みが必要不可欠である。

本稿は数学的証明を含まない設計仕様である。先行論文群 [W1]–[W11], [M1]–[M3], [補講1]–[補講4], [思考実験_R軸] において暗黙に前提とされている粒子のデータ構造を、型付き整数演算の設計仕様として明示的に定義する。

### v2.0 からの改訂（v2.1）

**`δδt` フィールドの削除**：v2.0 で Particle 構造に追加した `δδt : DeltaT`（処理周期変化、加速度相当）を**削除**する。理由は以下の通り：

1. **可逆性の破れ**：`δt += δδt` を含む `step(p)` は、δt の符号反転で元の状態に戻らない（[W3] の情報保存要件に反する）
2. **保存則との非整合**：単一粒子で δt が変化することは、外因なく固有時の進み方が変わることを意味し、エネルギー・運動量保存則の暗黙の破れを引き起こす
3. **GR 自由粒子原理との整合**：自由粒子は測地線に従い、加速度効果は外力（相互作用）から生じる。単一粒子の基本演算で加速度を扱うべきではない

加速度効果は **すべて相互作用処理（粒子間または粒子-Mesh 間）の中で、保存則を満たす形で σ や δt が変化する**ことで表現される。詳細は別稿 D2「中心投影フレームワークの動力学規則仕様」（執筆中）を参照。

修正された Particle 構造は **4 フィールド** `(σ, V, δV, δt)` となる（§5）。Mesh 構造からも対応する `δδt_mesh` を削除（§7）。

## §1 int256 有界性と次元解釈

### §1.1 有界性要件の根拠（v1.0 から変更なし）

[W3] は万物の理論の構造要件として有界性を要求する。[M1] の三層構造 $R$–$R_1$–$R_0$ において、最大の曲率半径 $R_0$ は有限である。この有限性とプランクスケールの離散性から、全位相量の表現に必要なビット数の上界が導かれる。

### §1.2 必要ビット数

**空間軸**: 観測可能宇宙径 $D \approx 1.892 \times 10^{27}$ m、プランク長 $l_P = 1.616 \times 10^{-35}$ m から

$$n_{\text{space}} = D / l_P \approx 1.171 \times 10^{62}, \quad b_{\text{space}} = 207 \text{ bit}$$

**時間軸**: 宇宙時間 $T \approx 8.710 \times 10^{17}$ s、プランク時間 $t_P = 5.391 \times 10^{-44}$ s から

$$n_{\text{time}} = T / t_P \approx 1.616 \times 10^{61}, \quad b_{\text{time}} = 204 \text{ bit}$$

**結論**: 256ビット符号付き整数で空間 14 桁・時間 15 桁の余裕を持って表現可能。

### §1.3 自然単位系と次元

$c = 1$, $G = 1$（幾何学化単位系、$\hbar$ は採用しない）の下で

$$[L] = [T] = [M], \quad [L^{-1}] = [M^{-1}]$$

### §1.4 周波数解釈（v2.0 の中核改訂）

v1.0 では xyzt 軸の $[L^{-1}]$ を「波数」「加速度」と混在的に解釈していた。v2.0 では位相空間の自然な解釈として **「各軸方向の振動周波数」** に統一する。

| 物理量 | 次元 | v1.0 解釈 | **v2.1 解釈** |
|---|---|---|---|
| σ.k_x, k_y, k_z | $[L^{-1}]$ | 空間波数・加速度 | **各空間軸の振動周波数** |
| σ.k_t | $[L^{-1}]$ | 時間振動数 | **t 軸の振動周波数**（他軸と対等） |
| σ.R | $[L]=[M]$ | 曲率半径 | **スケール量・対応周期**（双対） |
| σ.Q | $[1]$ | 内部対称性 | 内部対称性（変更なし） |
| δt | $[L]=[M]$ | 固有時間ステップ | **処理ステップ周期**（時間ではない） |
| ~δδt~ | ~$[L]=[M]$~ | ~（v1.0 になし）~ | ~v2.0 で追加、v2.1 で削除（保存則・可逆性に反するため）~ |

### §1.5 周波数解釈の優位性

**(1) 6軸対称性の完全保持**: 位置型 4 軸（xyzt）はそれぞれ独立な振動モードを持ち、t 軸も他軸と対等。

**(2) 連続体力学概念の不要化**: 「加速度」は位置の二階時間微分という連続体概念であり、離散整数演算の枠組みには馴染まない。「振動周波数」は位相空間の自然な離散量。

**(3) 加速度効果は相互作用として処理**: 加速度的な効果（重力場・慣性系の影響）は単一粒子では表現せず、**すべて相互作用処理**（粒子間または粒子-Mesh 間）の中で σ や δt の変化として保存則整合的に扱う。これは GR の自由粒子原理（自由粒子は測地線に従う）と整合する。詳細は別稿 D2 を参照。

**(4) 周波数 × 周期 = 位相**: $\sigma.k \cdot \delta t$ が無次元の位相増分となり（$[L^{-1}] \cdot [L] = [1]$）、位相発展則が型整合する。

**(5) 特殊相対論の自動内包**: dispersion relation $\sigma.k_t^2 = \sigma.k_x^2 + \sigma.k_y^2 + \sigma.k_z^2 + m^2$ が周波数間の整数関係として自然に書ける。

```
型名         次元       値域                      用途
L256         [L]=[M]   |v| ≤ 2^255 - 1          長さ・時間・質量・周期
Linv256      [L⁻¹]     |v| ≤ 2^255 - 1          周波数・スケール逆数
Scalar256    [1]       |v| ≤ 2^255 - 1          無次元量
FullPhase    [1]       0 ≤ v ≤ P                0–720°
```

## §3 SignVector の定義（周波数解釈版）

### §3.1 構造

```
SignVector = (
    x : SpatialFreq,    -- x 軸の振動周波数 [L⁻¹]
    y : SpatialFreq,    -- y 軸の振動周波数
    z : SpatialFreq,    -- z 軸の振動周波数
    t : TemporalFreq,   -- t 軸の振動周波数 [L⁻¹]（他軸と対等）
    R : CurvatureR,     -- スケール量・周期 [L]
    Q : ColorQ,         -- 内部対称ラベル [1]
)
```

### §3.2 物理的意味

各 σ 成分は「該当軸での振動周波数」または「スケール量」：

- σ.k_axis = 0: 該当軸は振動しない（その方向の構造を持たない）
- σ.k_axis ≠ 0: 該当軸が周波数 |k_axis| で振動する
- 符号 sign(k_axis): 振動の位相方向（+ なら時計回り、− なら反時計回り）
- |k_axis|: 振動周波数の大きさ（離散整数値）

### §3.3 不変量

SignVector の値は相互作用イベントで変化するが、特定の組み合わせは安定モードを定義する：

- $\kappa_s = |\{i \in \{x,y,z\} : k_i \neq 0\}|$ — 空間非ゼロ数
- $\kappa_t = (1 \text{ if } k_t \neq 0 \text{ else } 0)$ — 時間非ゼロ指標
- $\kappa = \kappa_s + \kappa_t \in \{0, 1, 2\}$ — [W10] 命題2.1: $\kappa \geq 3$ は不安定

### §3.4 dispersion relation

質量殻条件は周波数間の整数関係として直接表現：

$$\sigma.k_t^2 = \sigma.k_x^2 + \sigma.k_y^2 + \sigma.k_z^2 + m^2$$

- $m = 0$（光子・グルーオン）: $k_t^2 = k_x^2 + k_y^2 + k_z^2$
- $m \neq 0$（質量粒子）: 差分の平方根が質量に対応

これにより特殊相対論が自動的に枠組みに含まれる。

### §3.5 W8 表 A との対応（v1.0 から継続）

[W8] 表 A の符号パターン $\{+, -, 0\}^6$ が SignVector の符号と対応する。整数値（軸スケール値）は [質量構造] §1 の $v_i$ に対応。

代表粒子：

| 粒子 | 集合 $(x,y,z,t,R,Q)$ | $\kappa$ | 分類 |
|---|---|---|---|
| 光子 $\gamma$ | $(0,0,0,0,+,0)$ | 0 | ゲージボソン（自由・球状波） |
| グルーオン $g$ | $(0,0,0,0,0,Q_i \to Q_j)$ | 0 | ゲージボソン |
| ヒッグス $H$ | $(0,0,+,0,0,0)$ | 1 | スカラー（縦波解釈は §7） |
| $W^+$ | $(+,0,0,0,+,0)$ | 1 | ゲージボソン |
| $Z^0$ | $(0,+,0,0,+,0)$ | 1 | ゲージボソン |
| 重力子 | $(0,0,0,\pm,\pm,0)$ | 1 | テンソルボソン |
| 電子 $e^-$ | $(+,0,0,+,0,0)$ | 2 | フェルミオン |

## §4 OneDWave の定義

### §4.1 構造（v1.0 から変更なし）

```
OneDWave = (
    θ₀ : CyclePhase,   -- 位置位相 [1]      (S1 §2.1, W9 §8.2)
    A  : Scalar256,     -- 振幅 [1]          (S1 §2.1, W9 §8.2)
    ℓ  : DeltaPhase,    -- 広がり位相 [1]    (W9 §8.3)
)
```

### §4.2 物理的意味

各粒子は非零位置型軸ごとに 1 つの OneDWave を持つ（最大 $\kappa$ 個）。各 OneDWave はその軸方向の振動の現在位相状態を保持する。

- $\theta_0$: 振動の現在位相（自由パラメータ、時間発展で変化）
- $A$: 振幅（自由パラメータ）
- $\ell$: 振動の局在広がり（SignVector から従属、[W9] §8.3）

## §5 Particle の定義（v2.1：4 フィールド構造）

### §5.1 構造

```
Particle = (
    σ   : SignVector,    -- 個性（各軸の振動周波数）
    V   : OneDWave[κ],   -- 各非零位置型軸の現在位相状態
    δV  : OneDWave[κ],   -- 1 ステップでの位相変位（v2.0 で追加）
    δt  : DeltaT,         -- 現在の処理ステップ周期
)
```

**v2.0 からの変更**: `δδt : DeltaT` フィールドを削除（5 フィールド → 4 フィールド）。理由は §0 の改訂説明を参照。

### §5.2 各フィールドの物理的意味

#### `σ : SignVector`

粒子の個性。各軸でどの周波数で振動するか。相互作用イベントで値が変化する（消滅＋生成として記述、しかしデータ構造としては σ の値が更新される）。

#### `V : OneDWave[κ]`

各非零位置型軸の現在位相状態。配列長 $\kappa$ は SignVector の非零位置型軸数（$\leq 2$）。各 V[j] は対応軸の振動の (位置位相, 振幅, 広がり) を保持する。

インデックス対応：

```
σ.k_x ≠ 0 のとき V に wave_x が含まれる
σ.k_y ≠ 0 のとき V に wave_y が含まれる
σ.k_z ≠ 0 のとき V に wave_z が含まれる
σ.k_t ≠ 0 のとき V に wave_t が含まれる
```

#### `δV : OneDWave[κ]` (v2.0 で追加)

V と同型の差分構造。1 処理ステップで V がどう変化するかを保持する：

```
δV[j] = (
    Δθ₀ : DeltaPhase,    -- 位相変位
    ΔA  : Scalar256,      -- 振幅変位
    Δℓ  : DeltaPhase,     -- 広がり変位
)
```

自由粒子の場合、δV[j].Δθ₀ は σ から計算可能：

$$\delta V[j].\Delta\theta_0 = \sigma.k_{a(j)} \cdot \delta t$$

ただし相互作用イベントでは δV が独立に変更されうる。

#### `δt : DeltaT`

**処理ステップ周期**（時間ではない）。普遍的な演算カウンタ。

- $\delta t > 0$: 順方向演算（因果の順方向）
- $\delta t < 0$: 逆向き演算（可逆性の保証）
- $\delta t = 0$: 静止

時間として観測されるのは V[t-軸].θ₀ の累積であり、δt 自身ではない（§6.4 参照）。

#### ~`δδt : DeltaT`（v2.1 で削除）~

v2.0 では Particle 構造に `δδt` フィールドを含めていたが、v2.1 で削除した。理由：

- **可逆性違反**: `step(p)` 内で `δt += δδt` を実行すると、δt の符号反転による逆演算で元の状態に戻らない
- **保存則違反**: 単一粒子で δt が変化することは外因なきエネルギー・運動量変化を意味し、保存則と整合しない
- **GR 自由粒子原理**: 加速度効果は外力（相互作用）の結果であるべきで、自由粒子の基本演算では扱えない

加速度効果は §3 以降の相互作用処理（粒子間または粒子-Mesh 間）の中で、保存則を満たす形で σ や δt が変化することで表現される。詳細は別稿 D2 を参照。

### §5.3 自由粒子の動力学は δV で完結

物理法則が 2 階微分で閉じる（ニュートン力学・GR・量子力学）ことから当初は δδt が必要と考えたが、v2.1 では：

- 自由粒子は等速測地線運動のみ（GR 原理）
- 加速度は相互作用の結果
- 単一粒子の基本演算は **位相速度（δV）と現在位相（V）の更新のみ**

したがって **4 フィールド（σ, V, δV, δt）で自由粒子表現として完全**。

### §5.4 メモリフットプリント（v2.1）

```
σ      : 6 × 256 = 1,536 bit
V      : κ × 3 × 256 ≤ 1,536 bit (κ ≤ 2)
δV     : κ × 3 × 256 ≤ 1,536 bit
δt     : 256 bit
合計   ≤ 4,864 bit = 608 byte / 粒子
```

v2.0 の 640 byte から 32 byte 削減（δδt フィールド削除分）。v1.0 の 224–416 byte からは増加するが、依然として宇宙の任意の粒子を完全に表現する量として極めて少ない。

## §6 演算則

### §6.1 基本：1 処理ステップでの更新（v2.1）

基本演算 `step(p)` は次の 1 つの操作のみ：

```
function step(p : Particle | null) → Particle | null:
    if p == null:
        return null
    for j in range(p.κ):
        p.V[j].θ₀ += p.σ.k_{a(j)} · p.δt
    return p
```

各非零位置型軸 $j$ に対して：

$$V[j].\theta_0 \mathrel{+}= \sigma.k_{a(j)} \cdot \delta t$$

これだけ。`σ`, `δV`, `δt` は `step(p)` では一切更新されない。

**v2.0 からの変更**: v2.0 では `δt += δδt` を含めていたが、v2.1 で削除（§5.2 参照）。

### §6.2 型整合検査

```
σ.k_axis · δt : SpatialFreq × DeltaT 
              = Linv256 × L256 
              = Scalar256 [1]                ✓

V[j].θ₀ + (σ.k · δt) : CyclePhase + Scalar256
                     = CyclePhase [1] (mod P/2)  ✓
```

### §6.2.1 5 原則の充足（v2.1）

`step(p)` は処理関数の 5 基本原則をすべて満たす：

1. **null 透過性**: 関数の最初で明示的に処理 ✓
2. **in-place 更新**: V[j].θ₀ を直接更新、新しい Particle 非生成 ✓
3. **型整合性**: D1 §2.3 の規則に準拠（§6.2 参照） ✓
4. **INPUT/OUTPUT 入れ替え可能性**: 関数は `p` を読み書きする対称構造 ✓
5. **時間反転による可逆性**: δt の符号反転で完全に巻き戻る：

```
1 回目（δt = +δt₀）: V → V + σ.k · δt₀
δt 反転（δt → -δt₀）
2 回目（δt = -δt₀）: V → V + σ.k · δt₀ + σ.k · (-δt₀) = V  ✓
```

これは [W3] の情報保存要件を構造的に保証する。

### §6.3 偏光・複数軸位相差の表現

複数の非零軸を持つ粒子（例：偏光を持つ一方向光）：

```
σ_polarized_light = (+k_x, +k_y, 0, 0, +R_γ, 0)
V = [wave_x, wave_y]
```

偏光位相差として：

$$\Delta\varphi = V[0].\theta_0 - V[1].\theta_0 \in \text{CyclePhase}$$

| 偏光状態 | $\Delta\varphi$ |
|---|---|
| 線偏光（同方向） | $0$ |
| 線偏光（反方向） | $P/4 = 180°$ |
| 円偏光（左） | $+P/8 = +90°$ |
| 円偏光（右） | $-P/8 = -90°$ |
| 楕円偏光 | 任意の整数値 |

全て int256 整数で正確に表現される（$15°$ の整数倍は $P/48$ の整数倍として誤差なし）。

### §6.4 時間の取り扱い：δt は処理ステップ、時間は V[t].θ₀

**重要原則**: δt は時間ではなく、普遍的な処理ステップ。時間として観測されるのは粒子ごとの V[t-軸].θ₀ の累積値。

電子の例（σ.k_t ≠ 0）:

```
σ_electron = (+k_x, 0, 0, +k_t, 0, 0)
V = [wave_x, wave_t]
1 ステップ:
  V[0].θ₀ += σ.k_x · δt   -- x 軸位相が進む（運動量相当）
  V[1].θ₀ += σ.k_t · δt   -- t 軸位相が進む（「電子が経験する時間」）
```

光子の例（σ.k_t = 0）:

```
σ_photon = (+k_x, +k_y, 0, 0, +R_γ, 0)
V = [wave_x, wave_y]      -- t 軸成分なし
-- σ.k_t = 0 ゆえ V に t 軸 OneDWave は存在しない
-- 結果: 光子の固有時間ゼロが構造的に保証される
```

### §6.5 加速度効果は相互作用処理で扱う（v2.1）

v2.0 では「δδt は背景 R(r) や相互作用によって決まる」としていたが、v2.1 では：

- **加速度効果は単一粒子の基本演算では扱えない**（保存則・可逆性に反する、§5.2 参照）
- **すべての加速度効果は相互作用処理で実現**（粒子間または粒子-Mesh 間で σ や δt が交換される）

具体的な相互作用規則は別稿 D2 で定義する。重力時間遅れ・偏光回転・周波数シフトは、すべて相互作用イベントの集積として保存則整合的に表現される。

### §6.6 必要としない演算（v1.0 と同じ）

以下の演算は本仕様の範囲では必要としない：

1. **除算**: $R/R_1$ は $R \cdot R_1^{-1}$ として乗算に変換
2. **平方根**: 連続体的概念。離散系では整数値が直接的に物理量を決定
3. **三角関数**: 連続体 sine-Gordon の道具。離散変調表で代替可能
4. **浮動小数点演算**: 全フィールド int256、丸め誤差は構造的に発生しない

cos/sin が必要な箇所（例：縦波振動の変調）は、有限段階の離散変調表（lookup table）で代替する。

## §7 Mesh 構造の新設（v2.0 で追加）

### §7.1 縦波モードの分離

[W8] 表 A の解釈において、ヒッグス（$\sigma_H = (0, 0, +k_z, 0, 0, 0)$）は他のスピン1ボソンと異なり $\sigma.R = 0$ かつ $\text{spin} = 0$ である。これは構造的に「位置位相を振動させる横波粒子」ではなく、**「メッシュ自身の縦波振動」** として扱うのが自然である。

同様に、グラビトン（重力波）も時空計量の縦波的振動として、Mesh の振動モードと解釈できる。

これらの縦波モードを Particle として扱うと、データ構造に無理が生じる（OneDWave は本質的に位相回転を記述する型であり、軸方向の伸縮振動を直接表現できない）。本稿では **Mesh 構造を新設し、縦波モードを Particle と分離管理** する。

### §7.2 Mesh の構造（v2.1）

```
Mesh = (
    -- 背景スケール
    R_0           : L256,             -- 中心投影の中心長さ（背景）
    
    -- 単一軸縦波モード（spin 0、SO(3) 対称性破れ）
    H_x           : OneDWave,         -- x 軸縦波
    H_y           : OneDWave,         -- y 軸縦波
    H_z           : OneDWave,         -- z 軸縦波 = ヒッグス
    H_t           : OneDWave,         -- t 軸縦波
    
    -- 多軸縦波モード（spin 2、重力波の偏光）
    H_xy_anti     : OneDWave,         -- xy 反位相 = 重力波 + 偏光
    H_xy_quarter  : OneDWave,         -- xy 90° 位相 = 重力波 × 偏光
    H_yz_anti     : OneDWave,         -- yz 反位相
    H_xz_anti     : OneDWave,         -- xz 反位相
    
    -- 縦波モードの差分
    δH_x, δH_y, δH_z, δH_t            : OneDWave,
    δH_xy_anti, δH_xy_quarter         : OneDWave,
    δH_yz_anti, δH_xz_anti            : OneDWave,
    
    -- グローバル処理ステップ
    δt_mesh       : DeltaT,
)
```

**v2.0 からの変更**:

- `H_xyz_inphase`（xyz 同位相）を削除し、軸別単一軸モード `H_x, H_y, H_z, H_t` に置換（理由：xyz 等方解釈ではヒッグスの世代質量階層を説明できない、[W11] §8.5 参照）
- ヒッグスは `H_z`（z 軸縦波）として位置付けられる
- `δδt_mesh` を削除（理由：Particle と同じく可逆性・保存則のため）

### §7.3 縦波モードの物理同定

| モード | 振動パターン | 物理同定 | spin |
|---|---|---|---|
| H_z | z 軸縦波（SO(3) 破れ） | **ヒッグス** | 0 |
| H_x, H_y, H_t | 各軸縦波 | 物理同定未確定（候補） | 0 |
| H_xy_anti | x 伸び・y 縮みの反位相 | 重力波 + 偏光 | 2 |
| H_xy_quarter | xy 90° 位相回転 | 重力波 × 偏光 | 2 |
| H_yz_anti | yz 反位相 | 重力波（別偏光） | 2 |
| H_xz_anti | xz 反位相 | 重力波（別偏光） | 2 |

**ヒッグス = z 軸縦波（H_z）**: W8 表 A の $\sigma_H = (0, 0, +k_z, 0, 0, 0)$ における z 軸選択を SO(3) 対称性破れの方向として保持する。第 3 世代フェルミオンが z 軸を共有することで最強結合 → 最大質量を獲得する（[W11] §8 と整合）。

**重力波の 2 偏光 (+, ×)**: 多軸縦波モード（H_xy_anti, H_xy_quarter, etc.）として表現。SO(3) 対称性は保たれる。

### §7.4 Mesh と Particle の結合（v2.1）

粒子の振動周波数 σ.k は Mesh の状態によって実効的に変調される。質量生成・周波数シフトはこの変調を介した相互作用として表現される（詳細は別稿 D2 参照）。

実効周波数（質量生成の例）：

$$\sigma.k_{\text{effective}}(particle, mesh) = \sigma.k(particle) - \delta k_H(\text{Mesh.H\_z VEV})$$

z 軸を共有する第 3 世代フェルミオンが H_z VEV と最強結合し、最大質量を獲得する（[W11] §8 と整合）。

### §7.5 動力学（v2.1：基本演算は step のみ）

Mesh の更新：

```
For each longitudinal mode H_*:
    H_*.θ₀ += (固有周波数) · δt_mesh
```

Particle の更新（Mesh と独立、基本演算は step(p) のみ）：

```
For each particle p:
    step(p)   -- §6.1 の通り、V のみ更新
```

Mesh-Particle 間の結合（質量生成・重力波など）は単一粒子の `step(p)` ではなく、相互作用処理（D2 で定義）の中で σ や δt を変化させる形で実現する。

### §7.6 メモリフットプリント（v2.1）

```
R_0    : 256 bit
H_*    : 8 × 3 × 256 = 6,144 bit  -- 単一軸4 + 多軸4 = 8 モード
δH_*   : 8 × 3 × 256 = 6,144 bit
δt_mesh : 256 bit
合計   = 12,800 bit = 1,600 byte / 全宇宙
```

宇宙全体で約 1.6 KB。Mesh モード数増加により v2.0 の 1 KB から増えたが、粒子数 $N$ に対して総メモリは $608N + 1600$ byte（粒子側は v2.0 から 32 byte 削減）。

## §9 v1.0 からの非互換性

### §9.1 解釈の変更

- σ.k は「波数・加速度」から「振動周波数」へ統一
- δt は「固有時間ステップ」から「処理ステップ周期」へ明示化
- 時間は V[t-軸].θ₀ として現れる派生量

### §9.2 構造の拡張と整理

**v2.0 で追加**（v2.1 で継続）:

- Particle に δV を追加（v1.0 の Particle はサブセットとして含まれる）
- Mesh 構造を新設（v1.0 にはなかった）

**v2.0 で追加し v2.1 で削除**:

- Particle の `δδt` フィールド（保存則・可逆性違反のため削除）
- Mesh の `δδt_mesh` フィールド（同上）
- Mesh の `H_xyz_inphase` モード（観測される世代質量階層を説明できないため、`H_z` 等の軸別モードに置換）

### §9.3 W8 表 A との整合

[W8] 表 A の 62 粒子分類は v2.1 でも有効。ヒッグスは **Mesh の H_z 励起**として位置付けられる（v2.0 で提案した H_xyz_inphase は v2.1 で訂正、[W11] §8.5 と整合）。標準模型 61 状態 + グラビトン 1 状態 = 62 という総数は変わらない。

## §10 結語

v2.1 の本質：

- **xyzt は周波数**（位相空間の自然な解釈）
- **6 軸完全対称**（位置型 4 軸 + R + Q）
- **横波（Particle）と縦波（Mesh）の分離**
- **基本演算 step(p) は V のみ更新**（5 原則すべて充足、加速度効果は相互作用処理に分離）
- **時間は V[t].θ₀**（位置位相として粒子ごとに進む）
- **全演算は int256 整数で閉じる**

この構造で動力学規則（D2、別稿）を書けば：

- 重力時間遅れ・偏光回転（粒子-Mesh 相互作用）
- ヒッグス機構（Mesh.H_z VEV と粒子の σ.k_z 結合）
- 重力波放射（Mesh.H_xy_anti モード励起）
- クーロン 2 体問題（粒子間相互作用）

が全て同じ枠組みで計算可能になる。すべての加速度効果は相互作用処理として保存則・可逆性を満たす形で実現される。

本稿は設計仕様であり、これらの動力学的応用は別稿に委ねる。

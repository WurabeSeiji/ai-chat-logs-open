## §0 本稿の位置づけ

本稿は数学的証明を含まない。本稿は、先行論文群 [W1]–[W11], [M1]–[M3], [S1]–[S2], [補講1]–[補講4] において暗黙に前提とされている粒子のデータ構造を、型付き整数演算の設計仕様として明示的に定義するものである。

本稿の主張は以下に限定される:

1. 宇宙の有界性とプランクスケールの離散性から、全位相量が256ビット符号付き整数で表現可能であること（§1）
2. 次元付きメタ型（`L256`, `Linv256`, `Scalar256`）により、全演算の次元整合性が型レベルで検証可能であること（§2）
3. 粒子の完全な状態が `SignVector`（静的ラベル）+ `OneDWave`（動的状態）の2層構造で過不足なく記述できること（§3, §4）
4. 本稿で定義する演算が整数の加算・乗算のみで閉じ、除算・平方根・三角関数を必要としないこと（§5）

**本稿は粒子の静的データ構造の定義である。演算規則（位相発展、相互作用等）の定義は含まない。** §5 は型検査の例示として演算を参照するが、演算そのものの定義は別稿に委ねる。

## §1 int256 有界性の導出

### §1.1 有界性要件の根拠

[W3] は万物の理論の構造要件として**有界性**を要求する。[M1] の三層構造 $R$–$R_1$–$R_0$ において、最大の曲率半径 $R_0$（我々の主観空間）は有限である。この有限性と、プランクスケールの離散性を組み合わせることで、全位相量の表現に必要なビット数の上界が導かれる。

### §1.2 空間軸の必要ビット数

**上界**: 観測可能な宇宙の直径を $D \approx 2 \times 10^{11}$ 光年と見積もる。

$$D = 2 \times 10^{11} \times 9.461 \times 10^{15} \;\text{m} \approx 1.892 \times 10^{27} \;\text{m}$$

**下界**: プランク長 $l_P = 1.616 \times 10^{-35}$ m を位相の最小分解能とする。

**分割数**:

$$n_{\text{space}} = \frac{D}{l_P} = \frac{1.892 \times 10^{27}}{1.616 \times 10^{-35}} \approx 1.171 \times 10^{62}$$

**必要ビット数**:

$$b_{\text{space}} = \lceil \log_2(n_{\text{space}}) \rceil = \lceil 206.2 \rceil = 207 \;\text{bit}$$

### §1.3 時間軸の必要ビット数

宇宙の時間スケールを $T \approx 2 \times 13.8 \times 10^{9}$ 年 $\approx 8.710 \times 10^{17}$ s（将来の膨張を含む余裕）、プランク時間 $t_P = 5.391 \times 10^{-44}$ s とすると:

$$n_{\text{time}} = \frac{T}{t_P} \approx 1.616 \times 10^{61}$$

$$b_{\text{time}} = \lceil \log_2(n_{\text{time}}) \rceil = \lceil 203.3 \rceil = 204 \;\text{bit}$$

### §1.4 int256 の十分性

256ビット符号付き整数の最大値は:

$$2^{255} - 1 \approx 5.790 \times 10^{76}$$

これは空間分割数 $10^{62}$ に対して14桁、時間分割数 $10^{61}$ に対して15桁の余裕を持つ。

**結論**: 幾何学化単位系（$c = 1$, $G = 1$）において、全位相量は **256ビット符号付き整数（int256）** で表現可能である。以降、int256 を全フィールドの基本整数型として採用する。

### §1.5 位相の正規化

位相定数 $P$（720° に対応する整数）を §2.2 で定義する。$P$ は 720 の倍数であり、物理的に意味のある全ての角度（15° の倍数）が正確な整数として表現される。

## §2 次元付きメタ型の定義

### §2.1 幾何学化単位系と基本次元

$c = 1$, $G = 1$ を採用する（幾何学化単位系）。**$\hbar = 1$ は採用しない。**

$\hbar = 1$ を採用すると $[M] = [L^{-1}]$ となり、波数と質量が同一次元に強制される。これは本フレームワークにおいて質量が $R$ 軸（曲率半径）を経由して決定される構造（[M1], [質量構造]）を隠蔽し、次元解析が破綻する。

$G = 1$ の下では $[G] = [L^3 M^{-1} T^{-2}] = 1$ かつ $c = 1$（$[T] = [L]$）より:

$$[L \cdot M^{-1}] = 1 \quad \Longrightarrow \quad [M] = [L]$$

本稿では **$[M] = [L]$ と定義する**。

|          物理量          | SI 次元 | 幾何学化単位系（$c = G = 1$） |
| :----------------------: | :-----: | :---------------------------: |
|           長さ           |   [L]   |              [L]              |
|           時間           |   [T]   |              [L]              |
|           質量           |   [M]   |     [L]（$G = 1$ による）     |
|           波数           |  [L⁻¹]  |             [L⁻¹]             |
|          振動数          |  [T⁻¹]  |             [L⁻¹]             |
|       曲率半径 $R$       |   [L]   |  [L] = [M]（質量次元と一致）  |
|           面積           |  [L²]   |          [L²] = [M²]          |
| 波数の積 $k_i \cdot k_j$ |  [L⁻²]  |         [L⁻²] = [M⁻²]         |

### §2.2 スカラー型の定義

全フィールドの基底型は Int256（256ビット符号付き整数）である。以下の7つのスカラー型を定義する。

```
型名         次元     値域                              用途
------------ -------- --------------------------------- -------------------------
L256         [L]=[M]  |value| ≤ 2^255 - 1              長さ・時間・質量・曲率半径
Linv256      [L⁻¹]    |value| ≤ 2^255 - 1              波数・振動数
Scalar256    [1]      |value| ≤ 2^255 - 1              無次元量（波高・正規化値等）
FullPhase    [1]      0 ≤ value ≤ P                     0–720°（スピン2全域）
CyclePhase   [1]      0 ≤ value ≤ P/2                   0–360°（位置位相 θ₀）
DeltaPhase   [1]      -P/2 ≤ value ≤ P/2                ±360°（変位位相）
SpinPhase    [1]      value ∈ {0, P/4, P/2, P}          {0, 180, 360, 720}°のみ
```

**位相定数 $P$**:

$$P = \left\lfloor \frac{2^{255} - 1}{720} \right\rfloor \times 720$$

$P$ = 720°。1° = $P/720$、15° = $P/48$、90° = $P/8$、180° = $P/4$、360° = $P/2$ は全て正確な整数。

### §2.3 演算規則

**加算:**

|     左辺      |   +   |     右辺      |   →   |         結果型         |    備考     |
| :-----------: | :---: | :-----------: | :---: | :--------------------: | :---------: |
|    `L256`     |   +   |    `L256`     |   →   |         `L256`         |             |
|   `Linv256`   |   +   |   `Linv256`   |   →   |       `Linv256`        |             |
| `CyclePhase`  |   +   | `DeltaPhase`  |   →   |     `CyclePhase`       | mod $P/2$   |
| `CyclePhase`  |   +   | `Scalar256`   |   →   |     `CyclePhase`       | mod $P/2$   |
| `DeltaPhase`  |   +   | `DeltaPhase`  |   →   |     `DeltaPhase`       |             |
| `SpinPhase`   |   +   | `SpinPhase`   |   →   |     `SpinPhase`        | mod $P$     |
|    その他     |   +   |    その他     |   →   | **型エラー**（禁止）   |             |

**乗算:**

|    左辺     |   ×   |    右辺     |   →   |    結果型     |      次元       |
| :---------: | :---: | :---------: | :---: | :-----------: | :-------------: |
|   `L256`    |   ×   |   `L256`    |   →   |   `L2_256`    |  [L²] = [M²]   |
|  `Linv256`  |   ×   |  `Linv256`  |   →   |  `Linv2_256`  | [L⁻²] = [M⁻²]  |
|  `Linv256`  |   ×   |   `L256`    |   →   | `Scalar256`   |       [1]       |

**除算**: 使用しない。$R/R_1$ は $R \cdot R_1^{-1}$ として乗算に変換（[M1] §5）。

## §3 符号ベクトル成分型の定義

### §3.1 軸型分類（Axis-Type Symmetry）

[W8] v10 および [W11] により、6次元超直方体の6軸は以下の3つの軸型に分類される:

|           軸型           |      軸      | 対称性                   |   メタ型    |
| :----------------------: | :----------: | :----------------------- | :---------: |
| 位置型（Position-type）  | $x, y, z, t$ | 超直方体の組合せ論で対称 |  `Linv256`  |
| スケール型（Scale-type） |     $R$      | 中心投影の径方向         |   `L256`    |
|  カラー型（Color-type）  |     $Q$      | 離散有限群               | `Scalar256` |

以下、各軸の成分型を個別に定義する。

### §3.2 SpatialK（空間波数型）— $x, y, z$ 軸

```
型 SpatialK <: Linv256      -- 基底メタ型: [L⁻¹]
    次元    : [L⁻¹]
    物理的意味: 空間方向の波数（波長の逆数）
    値域    : { n ∈ Z  |  |n| ≤ 2^255 - 1 }
    零値の意味: 当該空間軸に位相変化なし（波が伝搬しない方向）
    符号の意味: sign(k_i) ∈ {+1, -1, 0}
              +1 = 正方向伝搬, -1 = 負方向伝搬, 0 = 不活性
    先行定義: [W8] §1.3 の符号 ∈ {+, -, 0}、[W10] §2 の波動ベクトル成分
```

### §3.3 TemporalK（時間振動数型）— $t$ 軸

```
型 TemporalK <: Linv256     -- 基底メタ型: [L⁻¹]（c=1 により [T⁻¹] = [L⁻¹]）
    次元    : [L⁻¹]
    物理的意味: 時間方向の角振動数
    値域    : { n ∈ Z  |  |n| ≤ 2^255 - 1 }
    零値の意味: 時間発展を持たない（定常波、[W10] §5）
    符号の意味: sign(k_t) ∈ {+1, -1, 0}
              +1 = 未来方向, -1 = 過去方向, 0 = 時間不変
    先行定義: [W10] §2.1 の時間成分 k_t
```

**SpatialK との型区別**: `SpatialK` と `TemporalK` は同じメタ型 `Linv256` の派生型であるが、**相互に代入不可**とする。$x$ 軸に `TemporalK` 値を代入すること、$t$ 軸に `SpatialK` 値を代入することは型エラーである。これにより 3+1 分解（[W10] §2.1）が型レベルで保証される。

**演算互換性**: `SpatialK × TemporalK → Linv2_256 [M⁻²]` の乗算は許可される（符号付き面積の計算に必要）。加算は禁止（`SpatialK + TemporalK → 型エラー`）。

### §3.4 CurvatureR（曲率半径型）— $R$ 軸

```
型 CurvatureR <: L256        -- 基底メタ型: [L] = [M]
    次元    : [L] = [M]（G=1 による）
    物理的意味: 中心投影の曲率半径 / ポテンシャル / 加速度的量
    値域    : 符号付き平方数（[M1] §4 による制約）
    零値の意味: R=0 縮退（情報は保持、中心投影による [M1] §5）
    符号の意味: sign(R) ∈ {+1, -1, 0}
              符号は配向（内向き/外向き）を表す
    絶対値の意味: |R| = 曲率半径の大きさ（平方数）
    先行定義: [M1] §4 の三層構造 R–R₁–R₀
```

**SpatialK / TemporalK との型区別**: `CurvatureR` は `L256`（$[L] = [M]$）であり、`SpatialK` / `TemporalK`（`Linv256` $[L^{-1}]$）とは**逆次元**。加算は型エラー。乗算は:

```
CurvatureR × SpatialK  → L256 × Linv256 = Scalar256 [1]  （無次元化）
CurvatureR × CurvatureR → L2_256 [L²] = [M²]             （質量²）
```

### §3.5 ColorQ（カラーラベル型）— $Q$ 軸

```
型 ColorQ <: Scalar256       -- 基底メタ型: [1]（無次元）
    次元    : [1]（無次元）
    物理的意味: 内部対称性のカラーラベル
    値域    : 有限集合（有限群の元、具体的な状態数は [W8] で議論）
    零値の意味: カラー中性
    演算    : XOR（排他的論理和）/ 有限群の群演算
    先行定義: [W8] §1.3 の Q 軸

    制約:
      (C1) 有限群制約: Q の取りうる値は有限個
      (C2) 群演算の閉包: Q₁ ⊕ Q₂ ∈ ColorQ（群演算の結果も ColorQ 型）
      (C3) Q 軸は時空軸 (x,y,z,t) と独立に作用（[W11] 定義2.1 の軸型分類）
```

**他の全型との型区別**: `ColorQ` は無次元だが `Scalar256` の一般的な無次元量（位相角等）とは**意味論的に異なる**。`ColorQ` と `Scalar256` 間の加算・乗算は禁止。`ColorQ` 同士の演算は群演算（XOR 等）のみ。

### §3.6 SignVector 型定義

```
SignVector = (
    x : SpatialK,    -- [L⁻¹]  空間波数 k₁
    y : SpatialK,    -- [L⁻¹]  空間波数 k₂
    z : SpatialK,    -- [L⁻¹]  空間波数 k₃
    t : TemporalK,   -- [L⁻¹]  角振動数 k_t
    R : CurvatureR,  -- [L]=[M] 曲率半径（符号付き平方数、§3.4）
    Q : ColorQ,      -- [1]     カラーラベル（有限群, XOR）
)
```

**先行定義**: [W8] §1.3 の $(x, y, z, t, R, Q) \in \{+, -, 0\}^6$。本稿では符号 $\{+,-,0\}$ と絶対値（軸スケール値 [質量構造] §1）を単一フィールドに統合。

**軸型分類**（[W11] 定義2.1）:

```
位置型   = {x, y, z, t} : Linv256    -- 超直方体の組合せ論で対称
スケール型 = {R}          : L256       -- 中心投影の径方向（逆次元）
カラー型  = {Q}           : ColorQ     -- 離散有限群（無次元）
```

**不変オブジェクト**: 粒子のアイデンティティは SignVector で定義される。生成後のフィールド変更 = 別の粒子。相互作用による変更は消滅＋生成として扱う（[W11]）。

このため、以下は不変量となる（生成時に確定、以後不変）:

```
κ_s = |{i ∈ {x,y,z} : i ≠ 0}|      -- 空間非ゼロ数
κ_t = (1 if t ≠ 0 else 0)            -- 時間非ゼロ指標
κ   = κ_s + κ_t ∈ {0, 1, 2}          -- [W10] 命題2.1: κ≥3 は不安定
χ   = sign(k_s₀ · k_t) ∈ {+1, -1}   -- カイラリティ（κ=2 時のみ）
```

**制約**: [W8] 表A の62パターンに一致。

**メモリ**: 6 × 256 bit = 1,536 bit = 192 byte。

### §3.7 先行論文との対応

[W8] §1.3 において符号ベクトルは $(x, y, z, t, R, Q) \in \{+, -, 0\}^6$ と定義された。本稿では、この **符号**（$\{+, -, 0\}$）と **値**（int256）を統合して扱う:

- 符号 $\mathrm{sign}(k_i) \in \{+1, -1, 0\}$: 粒子の**分類ラベル**（[W8] 表A の62粒子分類を決定）
- 絶対値 $|k_i|$: 軸スケール値（[質量構造] §1 の $v_i$ に対応）

この統合により、[W8] の符号分類と [質量構造] の軸スケール値が **単一のフィールド** で表現される。

```
σ_photon = SignVector(
    x = 0,       -- SpatialK   不活性
    y = 0,       -- SpatialK   不活性
    z = 0,       -- SpatialK   不活性
    t = 0,       -- TemporalK  不活性
    R = +R_γ,    -- CurvatureR R > 0（ゲージボソン）
    Q = 0,       -- ColorQ     カラー中性
)
κ_s = 0, κ_t = 0, κ = 0（球状波）
R ≠ 0 がゲージボソンの特徴
位置型軸が全て 0 → 符号付き面積 = 0 → m = 0
```

**展開例 — トップクォーク $t$**:

```
σ_top = SignVector(
    x = 0,       -- SpatialK   不活性
    y = 0,       -- SpatialK   不活性
    z = +k_z,    -- SpatialK   第3世代: |k_z| = v_z（最大）
    t = +k_t,    -- TemporalK  未来方向
    R = 0,       -- CurvatureR 縮退
    Q = q,       -- ColorQ     q ∈ {R, G, B}
)
κ_s = 1, κ_t = 1, κ = 2（方向制約波）
符号付き面積: M(z,t) = k_z · k_t  [M⁻²]
|k_z| >> |k_x|, |k_y| → 最大の符号付き面積 → 最重量フェルミオン
```

### §3.9 導出量

符号ベクトルから以下の量が導出される。これらは独立フィールドではなく、計算により取得される:

**(D1) 伝搬速度**（[W10] §3.2）:

方向制約波（$\kappa_s = 1, \kappa_t = 1$）に対して:

$$v = \frac{k_{s_0}}{k_t} \quad (\text{SpatialK} / \text{TemporalK} = \text{Scalar256} \;[1])$$

ただし実装上は除算を避け、$(k_{s_0}, k_t)$ のペアを `(SpatialK, TemporalK)` として保持する。

**(D2) 符号付き面積**（[補講1] 定義1.2）:

$$M(i, j) = k_i \cdot k_j \quad (\text{SpatialK} \times \text{TemporalK} = \text{Linv2\_256} \;[L^{-2}] = [M^{-2}])$$

**(D3) カイラリティ**（[W10] 定義6.1）:

$$\chi = \mathrm{sign}(k_{s_0} \cdot k_t) \in \{+1, -1\}$$

**(D4) 安定モード判定**（[W10] 命題2.1）:

$$\kappa_s = |\{i \in \{x, y, z\} : k_i \neq 0\}| \leq 2$$
$$\kappa_t = \begin{cases} 1 & (k_t \neq 0) \\ 0 & (k_t = 0) \end{cases}$$
$$\kappa = \kappa_s + \kappa_t \in \{0, 1, 2\} \quad (\kappa \geq 3 \text{ は不安定})$$

**(D5) スピン**（[W10] §5, §6）:

$$\mathrm{spin}(\sigma) \to \text{SpinPhase} \in \{0, P/4, P/2, P\}$$

SignVector の軸パターンから一意に決定される。

## §4 OneDWave（1次元波）の定義

### §4.1 型定義

```
OneDWave = (
    θ₀ : CyclePhase,   -- [1]  位置位相（最大波高位置）  (S1 §2.1, W9 §8.2)
    A  : Scalar256,    -- [1]  最大波高（スカラー）      (S1 §2.1, W9 §8.2)
    ℓ  : DeltaPhase,   -- [1]  広がり位相               (W9 §8.3)
)
```

各粒子は非零軸ごとに1つの `OneDWave` を持つ。最大 $\kappa$ 個（§3.6）。$\kappa = 0$ の場合、$V$ は空配列となる。

### §4.2 命名の根拠

| D1 | 制約 | S1 §2.1 | W9 §8.2 | 自由/従属 |
|:---:|:---:|:---:|:---:|:---:|
| $\theta_0$ | `CyclePhase` | $\theta_0$（最大波高位置） | $\theta_0$（中心位相） | **自由** |
| $A$ | `Scalar256` | $A$（振幅） | $A$（最大波高） | **自由** |
| $\ell$ | `DeltaPhase` | — | $\sigma = \cos\alpha$（広がり） | SignVector から**従属**（[W9] §8.3） |

**$\ell$ の命名**: [W9] §8.3 は $\sigma$ と $\ell$ の両方を使用。本稿では $\sigma$ を SignVector の略記（§6.1）に使用しているため $\ell$ を採用。

### §4.3 次元検査

上記の定義において次元の矛盾がないことを確認する。

$$[\theta_0] = [\text{CyclePhase}] = [1]$$
$$[k_i \cdot \delta t] = [\text{Linv256}] \times [\text{L256}] = [L^{-1}] \times [L] = [1]$$

`CyclePhase + Scalar256 → CyclePhase`（mod $P/2$）✓

## §5 演算規則

### §5.1 固有時間ステップ δt

```
型 DeltaT <: L256           -- 基底メタ型: [L]（c=1 で [T] = [L]）
    次元    : [L] = [M]（G=1 による）
    値域    : { n ∈ Z  |  |n| ≤ 2^255 - 1 }
```

$\delta t$ は粒子ごとの固有時間ステップであり、任意の整数値をとる:

- $\delta t > 0$: 順方向演算（因果の順方向）
- $\delta t < 0$: 逆向き演算（可逆性の保証、情報保存要件 [W3] による要請）
- $\delta t = 0$: 固有時間が変位しない

### §5.2 位相発展（基本演算）

粒子 $p$ の各非零軸 $j \in \{0, \ldots, \kappa-1\}$ に対する位相発展:

$$p.V[j].\theta_0 \;\mathrel{+}= p.\sigma.k_{a(j)} \cdot \delta t$$

ここで $a(j)$ は $V[j]$ に対応する軸（$\sigma$ の $j$ 番目の非零位置型軸）。

上記の定義において次元の矛盾がないことを確認する。

**型検査**:

```
p.V[j].θ₀      : CyclePhase  [1]
p.σ.k_{a(j)}   : SpatialK    [L⁻¹]  （または TemporalK）
p.δt            : DeltaT      [L]
k · δt          : SpatialK × DeltaT = Scalar256  [1]
θ₀ += (k·δt)   : CyclePhase + Scalar256 = CyclePhase  [1]  (mod P/2) ✓
```

**可逆性**: $\delta t < 0$ とすることで、任意のステップを巻き戻せる。全演算が加算（減算は負の加算）と乗算のみであり、情報の非可逆的損失は発生しない。

### §5.3 符号付き面積

粒子 $p$ のフェルミオン面 $(i, j)$ に対する符号付き面積:

$$M(i, j) = p.\sigma.k_i \cdot p.\sigma.k_j$$

上記の定義において次元の矛盾がないことを確認する。

**型検査**:

```
p.σ.k_i  : SpatialK   [L⁻¹] = [M⁻¹]
p.σ.k_j  : TemporalK  [L⁻¹] = [M⁻¹]
M(i,j)   : SpatialK × TemporalK = Linv2_256 [L⁻²] = [M⁻²]  （質量⁻²の次元）
```


### §5.4 R/R₁ 正規化（スケール変換）

[M1] の三層構造における $R/R_1$ 正規化:

$$\text{正規化値} = R \cdot R_1^{-1}$$

上記の定義において次元の矛盾がないことを確認する。

**型検査**:

```
R       : CurvatureR  [L] = [M]
R₁⁻¹   : Linv256     [L⁻¹]（予め逆数として保持）
R·R₁⁻¹ : CurvatureR × Linv256 = Scalar256  [1]  ✓  無次元
```

$R = 0$ の場合: $0 \cdot R_1^{-1} = 0$（`Scalar256` 型のゼロ）。情報は縮退として保持され、ゼロ除算は発生しない。

### §5.5 必要ない演算

以下の演算は本稿の基本演算では必要としない:

1. **除算**: $R/R_1$ は $R \cdot R_1^{-1}$ として乗算に変換（§5.4）
2. **平方根**: ローレンツ因子 $\sqrt{1 - v^2}$ は連続近似。離散系では波数の整数値が直接的に広がりを決定（§4.1）
3. **三角関数**: $\sin\alpha$, $\cos\alpha$ は連続体 sine-Gordon の道具。離散系では波数の整数比が速度・広がりを直接表現
4. **浮動小数点演算**: 全フィールドが int256 であり、丸め誤差は構造的に発生しない

## §6 粒子の完全な状態表現

### §6.1 Particle 型定義

```
Particle = (
    σ  : SignVector,     -- 静的ラベル（粒子が何者か）
    V  : OneDWave[κ],    -- 動的状態（長さ κ ∈ {0, 1, 2}）
    δt : DeltaT,         -- 固有時間ステップ
)
```

配列 `V` の長さ $\kappa$ は SignVector の非零位置型軸の数（§3.6、[W10] 命題2.1）であり、$\sigma$ 生成時に確定する。各要素は $\sigma$ の非零位置型軸に順対応する:

| $\kappa$ | 代表粒子 | V の内容 |
|:---:|:---|:---|
| 0 | 光子 $\gamma$, グルーオン $g$ | `V = []`（空） |
| 1 | ヒッグス $H$, $W^+$ | `V = [wave₁]` |
| 2 | 電子 $e^-$, トップ $t$ | `V = [wave₁, wave₂]` |

**具体例**:

```
-- 電子: σ = (+k_x, 0, 0, +k_t, 0, 0), κ = 2
V = [wave_x, wave_t]    -- V[0] → x 軸, V[1] → t 軸

-- ヒッグス: σ = (0, 0, +k_z, 0, 0, 0), κ = 1
V = [wave_z]             -- V[0] → z 軸

-- 光子: σ = (0, 0, 0, 0, +R_γ, 0), κ = 0
V = []                   -- 波動状態なし
```

### §6.2 メモリフットプリント

必要とされるメモリ量を概算する。

```
SignVector      = 6 × 256           = 1,536 bit = 192 byte
OneDWave (1要素) = 3 × 256           =   768 bit =  96 byte
δt              = 1 × 256           =   256 bit =  32 byte

Particle (κ 依存):
    κ = 0:  1,536 + 0 + 256       = 1,792 bit = 224 byte
    κ = 1:  1,536 + 768 + 256     = 2,560 bit = 320 byte
    κ = 2:  1,536 + 768×2 + 256   = 3,328 bit = 416 byte
```

## §8 結語

本稿は、中心投影フレームワークの粒子データ構造を、数学的記法ではなくプログラミング言語の型定義として記述した。連続・無限を前提とする数学的記法を使って、離散・有界・整数という本フレームワークを表現すると曖昧な表現が多く残るためである。

int256 の十分性（§1）、次元付きメタ型（§2）、SignVector と OneDWave の2層構造（§3, §4）、および演算規則（§5）により、粒子の完全な状態を有限ビットの型付き整数で過不足なく表現できることを示した。

本稿は証明論文ではなく設計仕様である。
